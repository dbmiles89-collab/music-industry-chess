## 18 Jul — Scaffolding React + Laravel
PHP is a back-end language for servers. Learning about the DOM in front-end web development, I understand that programs such as Lit and React were created to easily create and update DOM templates. These programs make it easier to translate what is wanted into plain JavaScript, doing the heavy lifting for the developer. This is similar to the relationship between PHP and Laravel on the back end. It is the same concept ontologically, even if it is different at a substrate level. Realizing this helped me understand frameworks as abstractions over a language's pain points across both tiers of the stack.
I used npm and Composer in the command line to fetch the scaffolding skeletons and download the initial libraries and commands I will need. Specifically, npm create vite and composer create-project fetched these skeletons.
Running npm run dev and php artisan serve launched the development servers.
This created two separate servers. I initially was confused as to why the front end and back end need separate servers, but I now understand that this is a scaffolding process for the development stage, not the app's final form. Front-end code does not actually need a server to execute. A server can simply hold JavaScript files react compiles as data and hand them over to a browser, which is what actually speaks and understands JavaScript.  However, in the beginning, it is much better to build them separately for clarity.
